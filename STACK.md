# Технологический стек

Telegram-бот «Рекомендатор книг» — асинхронное Python-приложение с ML-рекомендациями и локальной базой данных.

## Обзор

| Слой | Технология |
|------|------------|
| Язык | Python 3 |
| Telegram API | aiogram 3.x |
| База данных | SQLite + aiosqlite |
| ML / рекомендации | scikit-learn, NumPy |
| Конфигурация | python-dotenv |
| Форматы данных | JSON, CSV |

---

## Язык и рантайм

- **Python 3** — основной язык проекта
- **`asyncio`** — асинхронный цикл событий (`asyncio.run(main())`)
- **`logging`** — стандартный модуль логирования
- **`pathlib`, `os`, `csv`, `json`, `random`, `traceback`** — стандартная библиотека Python

---

## Telegram-бот: aiogram 3.15.0

Фреймворк для работы с [Telegram Bot API](https://core.telegram.org/bots/api).

| Компонент | Использование в проекте |
|-----------|-------------------------|
| `Bot`, `Dispatcher` | Инициализация бота и диспетчера (`main.py`) |
| `Router` | Модульные обработчики в `handlers/` |
| `Command`, `F` | Фильтры команд и callback-данных |
| FSM (`FSMContext`, `StatesGroup`) | Пошаговый опрос и админ-формы (`states/`) |
| `MemoryStorage` | Хранение FSM-состояний в памяти |
| `ReplyKeyboardMarkup`, `InlineKeyboardMarkup` | Клавиатуры (`keyboards/`) |
| `BaseMiddleware` | Авторизация админов и обработка ошибок (`middlewares/`) |
| `ParseMode.HTML` | Форматирование сообщений |
| `FSInputFile` | Отправка файлов (экспорт) |

Режим работы: **long polling** (`dp.start_polling(bot)`).

## База данных

| Технология | Версия | Назначение |
|------------|--------|------------|
| **SQLite** | встроенная | Файловая БД `data/books.db` |
| **aiosqlite** | 0.20.0 | Асинхронный доступ к SQLite |

### Таблицы

- `users` — пользователи и флаг администратора
- `books` — каталог книг
- `user_preferences` — предпочтения после опроса
- `read_history` — история прочитанного
- `book_grades` — оценки книг (1–5)
- `error_logs` — журнал ошибок

## ML и рекомендации

| Библиотека | Версия | Роль |
|------------|--------|------|
| **scikit-learn** | 1.5.2 | TF-IDF векторизация, косинусное сходство |
| **NumPy** | 2.1.3 | Зависимость scikit-learn |

Алгоритм (`services/recommender.py`):

1. Собирается текстовый профиль пользователя (жанры, настроение, интересы, объём).
2. Для каждой книги строится текстовое описание.
3. **TfidfVectorizer** преобразует тексты в векторы.
4. **cosine_similarity** ранжирует книги по близости к профилю.
5. Дополнительные фильтры: жанр, настроение, рейтинг, длина книги.

## Архитектура проекта

```
┌─────────────────────────────────────────┐
│              Telegram API               │
└──────────────────┬──────────────────────┘
                   │ long polling
┌──────────────────▼──────────────────────┐
│  aiogram (handlers, keyboards, FSM)     │
│  middlewares: auth, error handling      │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│  services (бизнес-логика)               │
│  ├── user_service                       │
│  ├── book_service                       │
│  └── recommender (scikit-learn)         │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│  aiosqlite → SQLite (data/books.db)     │
└─────────────────────────────────────────┘
```

### Слои

| Папка | Ответственность |
|-------|-----------------|
| `handlers/` | Команды и callback-обработчики |
| `keyboards/` | Reply и Inline клавиатуры |
| `states/` | FSM-состояния опроса и админки |
| `middlewares/` | Регистрация пользователей, права админа, лог ошибок |
| `services/` | Работа с БД, рекомендации, импорт/экспорт |
| `database/` | Подключение и схема SQLite |
| `utils/` | Форматирование сообщений |
| `data/` | БД, sample-книги, экспорт |

## Что не используется

- Веб-фреймворки (FastAPI, Flask, Django)
- ORM (SQLAlchemy, Tortoise)
- Внешние БД (PostgreSQL, Redis)
- Docker / CI/CD
- Внешние ML-сервисы или нейросети
- Redis для FSM (состояния хранятся в `MemoryStorage`)
