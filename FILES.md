# Описание файлов проекта

Краткое назначение каждого файла в `book_bot/`.

---

## Корень проекта

### `main.py`
Точка входа приложения. Инициализирует БД, при первом запуске заполняет каталог книгами-примерами, создаёт бота и диспетчер aiogram, подключает middleware и роутеры, запускает long polling.

### `config.py`
Загружает переменные окружения из `.env` (`BOT_TOKEN`, `ADMIN_IDS`). Хранит пути к папкам `data/` и `logs/`, путь к SQLite-файлу, а также справочники: жанры, настроения, варианты рейтинга, длины книг и интересы.

### `requirements.txt`
Список зависимостей Python: aiogram, aiosqlite, python-dotenv, scikit-learn, numpy.

### `.env`
Секреты и настройки окружения (не коммитится): токен бота и ID администраторов.

### `.gitignore`
Исключает из git: `.env`, `__pycache__`, скомпилированные `.pyc`, файл БД, экспорты и папку логов.

### `README.md`
Документация проекта: структура, установка, запуск, список команд пользователя и администратора, описание ML-рекомендаций.

### `STACK.md`
Описание технологического стека: язык, библиотеки, архитектура, зависимости.

---

## `database/` — работа с базой данных

### `database/db.py`
Подключение к SQLite через aiosqlite. Функция `init_db()` создаёт таблицы (`users`, `books`, `user_preferences`, `read_history`, `book_grades`, `error_logs`). `get_db()` возвращает асинхронное соединение.

### `database/__init__.py`
Реэкспортирует `init_db` и `get_db` для удобного импорта: `from database import init_db`.

---

## `handlers/` — обработчики Telegram-событий

### `handlers/__init__.py`
Собирает все роутеры в один через `setup_routers()` и возвращает главный `Router` для подключения в `main.py`.

### `handlers/start.py`
Команды `/start` и `/help`. Приветствие, список команд, показ меню (для админа — ещё и админ-клавиатура), запуск опроса.

### `handlers/survey.py`
Пошаговый опрос из 5 вопросов (FSM): жанры → настроение → рейтинг → объём → интересы. Обрабатывает inline-кнопки, сохраняет предпочтения в БД и вызывает рекомендации.

### `handlers/recommendations.py`
Выдача рекомендаций. Функция `send_recommendations()` запрашивает ML-подборку и форматирует ответ. Команды `/again` (новая подборка по сохранённым предпочтениям) и `/random` (случайные книги).

### `handlers/profile.py`
Команда `/profile` — показывает сохранённые предпочтения. Кнопка «Изменить предпочтения» перезапускает опрос.

### `handlers/history.py`
Команда `/history` — список прочитанных книг с датами. Вспомогательная функция `mark_book_as_read()` для отметки книги как прочитанной.

### `handlers/grade.py`
Команда `/grade` — оценка книги от 1 до 5. Пользователь вводит ID книги, затем выбирает оценку кнопкой. Оценка сохраняется, книга добавляется в историю прочитанного.

### `handlers/admin.py`
Админ-команды (доступны только `is_admin`):
- `/add_book` — пошаговое добавление книги
- `/delete_book` — удаление по ID
- `/update_book` — обновление поля книги
- `/create_admin`, `/drop_admin` — управление правами
- `/stats` — статистика бота
- `/export` — выгрузка каталога в JSON и CSV
- `/import` — загрузка книг из файла
- `/users` — список пользователей
- `/logs` — последние 10 ошибок

## `keyboards/` — клавиатуры Telegram

### `keyboards/user_keyboards.py`
Клавиатуры для пользователя:
- `main_menu_keyboard` — reply-меню с командами
- `genres_keyboard`, `interests_keyboard` — мультивыбор с галочками
- `mood_keyboard`, `rating_keyboard`, `length_keyboard` — шаги опроса
- `grade_keyboard` — кнопки 1–5 для оценки
- `profile_keyboard` — кнопка редактирования профиля

### `keyboards/admin_keyboards.py`
Reply-клавиатура админ-панели со всеми административными командами.

### `keyboards/__init__.py`
Реэкспорт всех функций клавиатур.

---

## `middlewares/` — промежуточные обработчики

### `middlewares/auth_middleware.py`
`AdminMiddleware` — при каждом запросе проверяет пользователя в БД, при первом визите создаёт запись (админ, если ID в `ADMIN_IDS`), передаёт флаг `is_admin` в обработчики.

### `middlewares/error_middleware.py`
`ErrorMiddleware` — перехватывает исключения, записывает их в таблицу `error_logs` (с ID пользователя и traceback), затем пробрасывает ошибку дальше.

### `middlewares/__init__.py`
Реэкспорт `AdminMiddleware` и `ErrorMiddleware`.

---

## `states/` — FSM-состояния

### `states/survey_states.py`
Группы состояний aiogram FSM:
- `SurveyStates` — шаги пользовательского опроса (5 состояний)
- `AdminStates` — шаги админ-форм (добавление/удаление/обновление книги, импорт, управление админами)
- `GradeStates` — выбор книги для оценки

### `states/__init__.py`
Реэкспорт `SurveyStates`, `AdminStates`, `GradeStates`.

---

## `services/` — бизнес-логика

### `services/user_service.py`
Работа с пользователями в БД:
- создание и получение пользователя, назначение админа
- сохранение и чтение предпочтений
- история прочитанного, оценки книг
- статистика и список активных пользователей

### `services/book_service.py`
CRUD для каталога книг:
- получение всех книг и книги по ID
- добавление, удаление, обновление поля
- импорт из JSON/CSV и экспорт в JSON/CSV

### `services/recommender.py`
ML-рекомендации. Строит текстовый профиль пользователя и описания книг, считает TF-IDF и косинусное сходство (scikit-learn), фильтрует по жанру, настроению, рейтингу и объёму, формирует топ-3 с объяснением «почему подходит». Режим `/random` — случайный выбор без ML.

### `services/__init__.py`
Реэкспорт публичных функций всех сервисов.

---

## `utils/` — вспомогательные функции

### `utils/messages.py`
Форматирование текста для Telegram (HTML):
- `format_book_recommendation` — карточка рекомендации с названием, автором, рейтингом и причиной
- `format_preferences` — читаемый вывод сохранённых предпочтений

### `utils/__init__.py`
Реэкспорт функций форматирования.

---

## `data/` — данные

### `data/sample_books.json`
Пример каталога из 3 книг в формате JSON. Справочный файл; при запуске `main.py` использует встроенный список из 12 книг (не этот файл напрямую).

### `data/books.db` *(создаётся при запуске)*
Файл SQLite-базы со всеми таблицами. Генерируется автоматически, не коммитится.

### `data/books_export.json`, `data/books_export.csv` *(создаются по команде /export)*
Файлы экспорта каталога, отправляемые админу.

---

## `logs/` *(создаётся при запуске)*

Папка для логов. Сейчас основной журнал ошибок хранится в таблице `error_logs` в БД; папка создаётся в `init_db()` на будущее.

---

## Схема связей

```
main.py
  ├── config.py
  ├── database/db.py
  ├── handlers/          → keyboards/, states/, services/, utils/
  ├── middlewares/
  └── services/
        ├── user_service.py   → database/
        ├── book_service.py   → database/
        └── recommender.py    → book_service, scikit-learn
```
